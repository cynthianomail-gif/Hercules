## 目的

讓企劃能在遊戲執行中即時調整兩類參數，不需改程式碼：

1. **節奏**：各段演出時長，預設值對齊節奏表 v2，可即時微調。
2. **佈局**：畫面物件的位置與尺寸，直接在畫布上拖曳調整。

調整結果自動保存於瀏覽器，並可匯出 JSON 交回工程端寫成正式預設值。

## 整體架構

- 畫面角落新增 **🔧 按鈕**（沿用現有 📊 / 📖 按鈕樣式），開啟「調校面板」。
- 調校面板為 DOM 浮動面板（與現有 `#infoPanel` 同模式），內分 **節奏** / **佈局** 兩分頁。
- 新增兩份集中設定物件，作為單一事實來源：
  - `TIMING`：所有演出時長，**單位秒**；遊戲內部換算為 60fps 幀數（`Math.round(秒 × 60)`）。
  - `LAYOUT`：所有可移動物件的座標／尺寸，預設值 = 目前程式內寫死的座標。
- 遊戲程式中散落的幀數魔術數字（如 `popWaitTimer = 30`、`killEffectTimer >= 40`、`chargeTimer = 35`）全部改為引用 `TIMING`；繪製座標改為引用 `LAYOUT`。

## 佈局分頁（位置編輯器）

切到佈局分頁即進入**編輯模式**；關閉面板或切回節奏分頁即離開。

### 可移動物件（註冊表）

| 物件 | 可調屬性 |
|---|---|
| 大怪獸插畫 ×4（左後紅龍、右後三頭巨人、左前野豬、右前亞馬遜） | x, y, w, h |
| 小怪血條/頭像錨點 ×4 | x, y |
| Boss 飾框內容（左/右：徽章、數字、血條群組） | x, y（群組整體位移） |
| 卡牌列 | x, y |
| 盤面（GRID） | x, y |
| 下方 UI 列（BET/SPIN 等） | y |
| JP 橫幅 | x, y |
| FG/BG 累積欄 | x, y |

### 互動

- 編輯模式下所有已註冊物件顯示**虛線外框＋名稱標籤**。
- 滑鼠點選 → 選中（外框變實線）→ 直接拖曳移動。
- 方向鍵 1px 微調；Shift+方向鍵 10px。
- 面板同步顯示選中物件的 x / y（/ w / h），可直接輸入數字。
- 編輯模式中點擊畫布**不觸發 spin**（輸入攔截優先給編輯器）。
- 拖曳範圍不設限（允許移出畫面外），可隨時「重設」救回。

## 保存與匯出

- 任何調整即時寫入 `localStorage`（鍵：`hercules_tuning`），重新整理／關閉重開皆保留。
- 儲存格式帶版本號：`{ version: 1, timing: {...}, layout: {...} }`。
- 兩分頁各有：
  - **重設為預設**：清除該分頁的自訂值，回到節奏表／原始座標。
  - **匯出 JSON**：將目前設定複製到剪貼簿，交回工程端寫成正式預設值。

## 邊界處理

- localStorage 資料解析失敗或版本不符 → 自動作廢，退回預設值（不報錯、不阻擋遊戲）。
- 滑桿限制 0～5 秒，避免調出卡死節奏；數字框輸入超界時鉗制到範圍內。
- 儲存的參數鍵若在新版程式中不存在（參數增減）→ 忽略未知鍵、缺漏鍵補預設值。

## 測試

- 以本地 dev server 開啟遊戲手動驗證：
  - 節奏：調整「每顆圖示得分演繹」等參數後 spin，確認演出時長變化；重設後回到節奏表值。
  - 佈局：拖曳大怪獸插畫與血條錨點，確認繪製與攻擊命中點同步移動；重新整理後位置保留。
  - 邊界：手動塞壞 localStorage 確認自動退回預設。

## 不做的事（YAGNI）

- 不做 Quick／Turbo 節奏（節奏表未定義）。
- 不做面板內直接改寫 index.html 的儲存機制（以匯出 JSON 取代）。
- 不做 FREE GAME 專屬節奏（節奏表標示待決策）。
